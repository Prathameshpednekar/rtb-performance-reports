Error Analysis: Problems Identified & Solutions Applied

During the 2M load test on the OLD server, 56,521 requests failed with a 2.83% error rate. Below is a detailed analysis of the errors encountered and how the New Optimized Server configuration resolved them.

⚠ Problems Found in OLD Server
1

Connection Refused Errors

Under heavy load (2M requests), the server started refusing new connections. The connection pool was exhausted, causing incoming requests to fail with "Connection refused" errors.

2

Socket Timeout Errors

Many requests timed out waiting for a response. The server was overwhelmed and could not process requests within the acceptable time window, leading to socket timeout exceptions.

3

Thread Pool Exhaustion

The worker thread pool was undersized for high concurrency. All available threads were busy processing requests, causing new requests to queue and eventually timeout.

4

Memory Pressure & GC Pauses

High memory usage triggered frequent garbage collection pauses, causing intermittent latency spikes and request failures during GC stop-the-world events.

✔ Solutions in New Optimized Server
1

Increased Connection Pool Size

Expanded the connection pool limits and configured connection keep-alive settings. This allows the server to handle more concurrent connections without refusing new requests.

2

Optimized Timeout Configuration

Fine-tuned socket and connection timeouts. Implemented connection reuse and reduced idle connection cleanup intervals to maintain healthy connection pools.

3

Expanded Worker Thread Pool

Increased worker thread count to match expected concurrency. Implemented async I/O where possible to reduce thread blocking and improve throughput capacity.

4

JVM & Memory Optimization

Tuned JVM heap settings, switched to G1GC garbage collector with optimized pause time goals. Reduced object allocation rate to minimize GC frequency and duration.
